## Rủi ro phát hiện

- **Schema lệch M2:** SQL gốc (`StudyMate_Schema.sql`) dùng `TutorProfileId` → `TutorProfiles`. M1-T9 dùng **`UserId` → AspNetUsers** để không bị block khi M2 chưa có. Khi làm M2 cần migration chuyển FK (hoặc map dual-support).
- Navbar / Profile **chưa có link** “Chứng chỉ” — Tutor phải gõ URL `/Account/Certificates`.
- Chưa có action xóa chứng chỉ / xóa file vật lý khi xóa record.
- `AiVerifyNote` luôn null lúc upload — đúng scope; Admin/AI (M8/M12) sẽ xử lý sau.
- File user upload trong `wwwroot/uploads/` **không commit** lên git (chỉ `.gitkeep`).

## Hiểu được gì

- Role-based auth: `[Authorize(Roles = "Tutor")]` + `AccessDeniedPath` cho UX 403 rõ ràng.
- Upload multipart cần `enctype="multipart/form-data"` và bind prefix `Upload.*`.
- Page VM (list + form) giúp redisplay lỗi validation mà vẫn giữ danh sách chứng chỉ.
- Model tạm `UserId` là trade-off hợp lý khi dependency module (TutorProfile) chưa sẵn — ghi rõ trong report để M2 không “quên migrate”.
